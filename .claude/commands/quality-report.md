# /quality-report — Daily Quality Check

일일 품질 점검. garden.sh + check_docs.sh + stale issue 탐지 + QUALITY_SCORE.md 갱신 후 리포트 PR 생성.

## 실행 순서

### 1. Worktree Setup

```bash
DATE=$(date +%Y-%m-%d)
YEAR=$(date +%Y)
MONTH=$(date +%m)
BRANCH="quality/report-${DATE}"
WORKTREE_PATH="worktrees/quality-${DATE}"

git fetch origin master

if [ ! -d "$WORKTREE_PATH" ]; then
  git branch -D "$BRANCH" 2>/dev/null || true
  git worktree add "$WORKTREE_PATH" -b "$BRANCH" origin/master
fi

mkdir -p "${WORKTREE_PATH}/docs/reports/${YEAR}/${MONTH}"

REPORT_FILE="${WORKTREE_PATH}/docs/reports/${YEAR}/${MONTH}/quality-${DATE}.md"
echo "Report will be written to: $REPORT_FILE"
echo "Worktree ready at: $WORKTREE_PATH"
echo "Branch: $BRANCH"
```

> **Note**: 각 Bash 호출은 독립 셸 — `DATE`, `YEAR`, `MONTH`, `WORKTREE_PATH`를 매 단계 재선언하고 `cd "$WORKTREE_PATH" && ...`로 실행할 것.

### 2. GP Drift Detection

```bash
DATE=$(date +%Y-%m-%d); WORKTREE_PATH="worktrees/quality-${DATE}"
cd "$WORKTREE_PATH" && bash scripts/clean/garden.sh --dry-run
```

GP-1~10 위반 캡처. `--dry-run`으로 자동 수정 없이 read-only.

### 3. Dead Links / Oversized Docs

```bash
DATE=$(date +%Y-%m-%d); WORKTREE_PATH="worktrees/quality-${DATE}"
cd "$WORKTREE_PATH" && bash scripts/clean/check_docs.sh --dry-run
```

데드 링크, 200줄 초과 문서, 누락 스펙 탐지.

### 4. Stale Issue Detection

```bash
DATE=$(date +%Y-%m-%d); WORKTREE_PATH="worktrees/quality-${DATE}"
cd "$WORKTREE_PATH" && gh issue list --repo yw0nam/DesktopMatePlus --state open --json number,title,createdAt --jq '.[] | "#\(.number) \(.title) (created: \(.createdAt[:10]))"'
```

14일 이상 open 상태 이슈 탐지.

### 5. GitHub Issues Health Check

```bash
DATE=$(date +%Y-%m-%d); WORKTREE_PATH="worktrees/quality-${DATE}"
cd "$WORKTREE_PATH"
gh issue list --repo yw0nam/DesktopMatePlus --state open --json number --jq 'length'
gh issue list --repo yw0nam/DesktopMatePlus --state open --label type:tech-debt --json number --jq 'length'
```

플래그 대상:
1. **Open issue 총 수** — 30개 초과 시 플래그
2. **Tech-debt 비율** — tech-debt이 전체의 50% 초과 시 플래그

리포트의 `## GitHub Issues Health` 섹션에 기록. 자동 수정 금지.

### 6. Quality Score Refresh

```bash
DATE=$(date +%Y-%m-%d); WORKTREE_PATH="worktrees/quality-${DATE}"
cd "$WORKTREE_PATH" && bash scripts/clean/garden.sh --metrics
```

`docs/QUALITY_SCORE.md` 등급 매트릭스 갱신.

### 7. Closed Issues Check

```bash
DATE=$(date +%Y-%m-%d); WORKTREE_PATH="worktrees/quality-${DATE}"
cd "$WORKTREE_PATH"
gh issue list --repo yw0nam/DesktopMatePlus --state closed --json number --jq 'length'
```

Closed 이슈 수 기록. 별도 조치 불필요.

### 8. Write Report

리포트를 `$REPORT_FILE` (Step 1에서 설정)에 작성:

```markdown
# Quality Report — YYYY-MM-DD

## GP Drift
[garden.sh --dry-run output summary]
- List each FAIL with [GP-N] (repo): description [file:line if available]

## Dead Links / Oversized Docs
[check_docs.sh output summary]
- List each FAIL with file path

## Stale Issues (2w+)
[Issues open for 14+ days]
- #N: description (created: YYYY-MM-DD)

## GitHub Issues Health
- Open issues: N (threshold: 30)
- Tech-debt ratio: N/M (threshold: 50%)

## Quality Score Update
[Paste updated QUALITY_SCORE.md table]

## Closed Issues
- Closed issues: N

## Violations Summary
- GP-3 (backend): N violations

## Recommendations
[Non-obvious quality patterns or systemic issues discovered]
```

### 9. Commit, Create PR, Clean Up

```bash
DATE=$(date +%Y-%m-%d); YEAR=$(date +%Y); MONTH=$(date +%m)
BRANCH="quality/report-${DATE}"; WORKTREE_PATH="worktrees/quality-${DATE}"
REPO_ROOT="$(pwd)"

cd "$WORKTREE_PATH"

git add docs/reports/ docs/QUALITY_SCORE.md

if git diff --cached --quiet; then
  echo "No changes to commit — skipping PR"
  cd "$REPO_ROOT"
  git branch -D "$BRANCH" 2>/dev/null || true
  git worktree remove "$WORKTREE_PATH" --force
  exit 0
fi

git commit -m "chore: quality report ${DATE}

Auto-generated by /quality-report command.
"
git push -u origin "$BRANCH" || { echo "Push failed — worktree preserved at $WORKTREE_PATH for retry"; exit 1; }

PR_URL=$(gh pr create \
  --base master \
  --title "chore: quality report ${DATE}" \
  --body "## Summary

Daily quality check on ${DATE}. See full report: \`docs/reports/${YEAR}/${MONTH}/quality-${DATE}.md\`

Auto-generated by /quality-report command.

🤖 Generated with [Claude Code](https://claude.com/claude-code)")

echo "PR created: $PR_URL"

cd "$REPO_ROOT"
git worktree remove "$WORKTREE_PATH" --force
git worktree prune
```

PR URL을 최종 응답에 포함할 것.

## 제약

- **소스 파일 수정 금지** — `docs/reports/YYYY/MM/`과 `docs/QUALITY_SCORE.md`만 쓰기 허용
- **garden.sh auto-fix** (ruff 등 Minor 위반)만 허용. 그 외 소스 수정 불가
